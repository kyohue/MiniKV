# Mini-KV 项目结构说明

## 📋 概述

本文档说明 Mini-KV 项目的目录结构和文件组织规则，帮助您快速了解项目布局。

---

## 📁 目录说明

### 1. `.kiro/` - Kiro 配置目录

存放 Kiro IDE 的配置文件和项目规范文档。

- `steering/protocol.md` - 项目开发协议，定义代码风格和开发规范
- `steering/file-organization.md` - 文件组织规则，定义文件放置标准

---

### 2. `include/` - 头文件目录

存放所有 C++ 头文件（`.h` 文件）。

**规则**：
- 所有类的声明都放在这里
- 使用 `#pragma once` 防止重复包含
- 只包含接口声明，不包含实现

**当前文件**：
- `MiniKV.h` - Database 类的头文件声明

---

### 3. `src/` - 源代码目录

存放所有 C++ 实现文件（`.cpp` 文件）。

**规则**：
- 类的实现文件与头文件同名
- 主程序入口文件以 `_main.cpp` 结尾
- 每个文件专注于单一职责

**当前文件**：
- `MiniKV.cpp` - Database 类的实现
- `cli_main.cpp` - 交互式 CLI 主程序

---

### 4. `tests/` - 测试代码目录

存放所有测试文件。

**规则**：
- 测试文件以 `test_` 开头
- 每个模块对应一个测试文件
- 使用自定义测试框架（纯 C++）

**当前文件**：
- `test_main.cpp` - 主测试套件（7 个测试用例）

---

### 5. `docs/` - 文档目录

存放所有项目文档，按类型分类。

#### 5.1 `docs/guides/` - 使用指南

用户使用手册和问题解决指南。

**当前文件**：
- `CLI_使用指南.md` - CLI 命令参考手册
- `中文乱码解决方案.md` - Windows 编码问题解决
- `项目结构说明.md` - 本文件

#### 5.2 `docs/phases/` - 阶段学习文档

各个开发阶段的详细学习指引。

**当前文件**：
- `Phase1_README_Guide.md` - Phase 1 学习指引

**未来文件**：
- `Phase2_README_Guide.md` - Phase 2 学习指引
- `Phase3_README_Guide.md` - Phase 3 学习指引
- `Phase4_README_Guide.md` - Phase 4 学习指引

#### 5.3 `docs/planning/` - 规划文档

项目规划和功能演进文档。

**当前文件**：
- `后续阶段CLI功能规划.md` - 详细的功能规划
- `CLI功能演进对比.md` - 各阶段功能对比表

---

### 6. `scripts/` - 构建和工具脚本

存放所有构建脚本和工具脚本。

**规则**：
- 构建脚本以 `build` 开头
- 清理脚本命名为 `clean.*`
- 同时提供 Windows（.bat）和 Linux/Mac（.sh）版本

**当前文件**：
- `build.bat` / `build.sh` - 编译测试版本
- `build_cli.bat` / `build_cli.sh` - 编译 CLI 版本
- `clean.bat` / `clean.sh` - 清理构建产物

---

### 7. `build/` - 构建输出目录

存放所有编译产物（已添加到 .gitignore）。

**内容**：
- `*.o` - 目标文件
- `*.exe` - 可执行文件（Windows）
- `mini_kv_test` / `mini_kv_cli` - 可执行文件（Linux/Mac）

**注意**：
- 此目录不应提交到版本控制
- 使用 `scripts/clean.*` 清理此目录

---

### 8. `data/` - 数据文件目录（Phase 2+）

存放数据库文件和备份（Phase 2 将启用）。

**规划内容**：
- `*.db` - 数据库文件
- `backups/` - 备份文件目录

---

## 🎯 文件命名规范

### 源代码文件

| 类型 | 命名规范 | 示例 |
|------|---------|------|
| 头文件 | `ClassName.h` | `MiniKV.h`, `Storage.h` |
| 实现文件 | `ClassName.cpp` | `MiniKV.cpp`, `Storage.cpp` |
| 主程序 | `purpose_main.cpp` | `cli_main.cpp`, `server_main.cpp` |
| 测试文件 | `test_module.cpp` | `test_main.cpp`, `test_storage.cpp` |

### 文档文件

| 类型 | 命名规范 | 示例 |
|------|---------|------|
| 使用指南 | `功能_使用指南.md` | `CLI_使用指南.md` |
| 阶段文档 | `Phase数字_描述.md` | `Phase1_README_Guide.md` |
| 规划文档 | 描述性名称 | `后续阶段CLI功能规划.md` |

### 脚本文件

| 类型 | 命名规范 | 示例 |
|------|---------|------|
| 构建脚本 | `build_purpose.*` | `build.bat`, `build_cli.sh` |
| 工具脚本 | `action.*` | `clean.bat`, `test.sh` |

## 💡 最佳实践

### 创建新文件时

1. 确定文件类型（源代码/文档/脚本）
2. 参考文件组织规则选择正确的目录
3. 遵循命名规范
4. 更新相关文档（如本文件）

### 修改项目结构时

1. 先更新 `.kiro/steering/file-organization.md`
2. 移动文件到新位置
3. 更新所有路径引用
4. 更新文档说明
5. 测试构建脚本是否正常工作

## 🎉 总结

Mini-KV 采用清晰的目录结构，将代码、文档、脚本分类存放：

- **代码**：`include/`, `src/`, `tests/`
- **文档**：`docs/guides/`, `docs/phases/`, `docs/planning/`
- **脚本**：`scripts/`
- **构建产物**：`build/`（不提交到版本控制）

这种结构便于：
- ✅ 快速定位文件
- ✅ 团队协作
- ✅ 项目维护
- ✅ 新人上手

遵循这些规则，可以保持项目的整洁和专业性！🚀
