## 🎨 CLI 提示符演进

### Phase 1（当前）
```
mini-kv>
```
- 简洁的提示符
- 只显示命令输入

### Phase 2
```
mini-kv>          # 无未保存修改
mini-kv[*]>       # 有未保存修改（需要 SAVE）
```
- 增加持久化状态指示
- `[*]` 表示有未保存的修改

### Phase 3
```
mini-kv[10]>      # 显示键值对数量
mini-kv[10][*]>   # 显示数量 + 未保存状态
```
- 显示当前键值对数量
- 方便快速了解数据库状态

### Phase 4
```
mini-kv[10|3]>           # 10 个键值对，3 个客户端
mini-kv(TX)[10|3]>       # 事务模式
mini-kv(WATCH)[10|3]>    # 监视模式
```
- 显示客户端数量
- 显示当前模式（事务、监视等）

## 🎯 实际使用场景演进

### Phase 1：学习和原型
```
mini-kv> SET config:app_name MyApp
mini-kv> GET config:app_name
"MyApp"
```
**适用场景**：
- 学习 C++ 基础
- 简单的配置管理
- 原型开发

---

### Phase 2：开发环境
```
mini-kv> SET user:1001:name 张三
mini-kv> SAVE
✓ 数据已保存

# 重启程序后
mini-kv> LOAD
✓ 从 mini_kv.db 加载了 1 个键值对
mini-kv> GET user:1001:name
"张三"
```
**适用场景**：
- 开发环境配置
- 临时数据存储
- 单机应用

---

### Phase 3：复杂应用
```
mini-kv> SETINT user:1001:score 100
mini-kv> INCR user:1001:score 50
150

mini-kv> SETLIST user:1001:tags C++ Database Learning
mini-kv> LLEN user:1001:tags
3

mini-kv> FILTER score > 120
找到 1 个匹配的键值对:
  [user:1001:score] => 150
```
**适用场景**：
- 游戏排行榜
- 用户标签系统
- 数据分析

---

### Phase 4：生产环境
```
# 客户端 1
mini-kv> WATCH balance
mini-kv> MULTI
mini-kv(TX)> SETINT balance 1000
mini-kv(TX)> EXEC
✓ 事务执行成功

# 客户端 2（同时操作）
mini-kv> MONITOR
[10:30:15] Client 1: WATCH balance
[10:30:16] Client 1: MULTI
[10:30:17] Client 1: SETINT balance 1000
[10:30:18] Client 1: EXEC

# 性能测试
mini-kv> BENCHMARK SET 10000
吞吐量: 79,681 ops/sec
```
**适用场景**：
- 高并发 Web 应用
- 分布式缓存
- 实时数据处理

## 💡 学习建议

### 循序渐进的学习路径

1. **Phase 1（1-2 天）**
   - 熟悉所有基础命令
   - 理解键值数据库概念
   - 掌握 CLI 的使用方法

2. **Phase 2（2-3 天）**
   - 学习持久化机制
   - 理解异常处理
   - 掌握数据备份策略

3. **Phase 3（3-5 天）**
   - 学习多类型支持
   - 掌握条件查询
   - 理解泛型编程

4. **Phase 4（5-7 天）**
   - 学习并发编程
   - 掌握事务机制
   - 理解性能优化

通过 4 个阶段的演进，Mini-KV 从一个简单的学习项目，逐步成长为一个功能完整的生产级数据库：

- **Phase 1**：9 个命令，基础功能 ✅
- **Phase 2**：13 个命令，持久化支持
- **Phase 3**：23 个命令，多类型支持

每个阶段都在 CLI 中有直观的体现，让您可以像使用 Redis 一样使用 Mini-KV！🚀
